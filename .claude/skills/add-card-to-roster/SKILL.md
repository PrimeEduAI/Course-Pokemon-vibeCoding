# 拍卡加入出戰名單

## Overview

你自己就有視覺能力：直接看圖辨識卡片上的寶可夢，**不需要**呼叫遊戲的掃卡 API、也不需要啟動 dev server。辨識出圖鑑編號後，用一行指令寫入收藏資料庫 —— 選角名單＝預設角色 ∪ 收藏，所以寫進收藏就等於加入可用角色。

## 步驟

1. **看圖辨識**：讀卡片上的寶可夢名稱（可能是中／英／日文），對照卡面圖案確認，得出**全國圖鑑編號**（1-1025）。
   - 卡上印的是哪隻就是哪隻：不要自己換成進化前／進化後。
   - MEGA／EX／V／VMAX 等特殊卡，一律用該寶可夢原始型態的圖鑑編號。
   - 沒把握時，列出 2-3 個候選（附中文名＋編號）問用戶，不要猜。
2. **寫入收藏**（在專案根目錄執行）：
   ```bash
   bun scripts/add-to-roster.ts <圖鑑編號> "<卡片上的名稱>" [照片檔路徑]
   ```
   - 名稱參數填**英文種名**（如 `"Charizard"`）：卡面是英文就抄卡名，中文／日文卡或無文字圖片就換成對應英文種名（腳本會自己補中文名）。
   - 用戶的圖有檔案路徑時把路徑帶上（相對或絕對路徑都可以，收藏大廳會顯示原照片）；純貼圖沒有路徑就省略第三個參數。
   - 重複執行安全：已在收藏會直接告知，不會加兩次。
3. **回報結果**：轉述腳本輸出 —— 中文名＋編號、戰鬥資料來源，以及：
   - 3D 模型**不在輕量素材包**時，一定要警告「選牠出戰畫面會出錯」並給解法（跑 `bash scripts/setup-assets.sh` 或改加別隻）。
   - 提醒：對戰頁的「選擇出戰寶可夢」要**重新進入**才會看到新角色。

## 常見錯誤

| 錯誤 | 修正 |
|---|---|
| 呼叫 `/api/scan` 或啟動 dev server 來辨識 | 不需要 —— 你直接看圖就能辨識（掃卡 API 是給遊戲內的洛托姆圖鑑頁用的） |
| 卡名是日文／德文等看不懂就放棄 | 以卡面圖案為主辨識，卡名當佐證 |
| 模型不在輕量包卻沒警告用戶 | 腳本會印 ⚠️ 警告，必須原樣轉述 |
| 辨識不確定還是直接加 | 先列候選問用戶，確認後再執行腳本 |
