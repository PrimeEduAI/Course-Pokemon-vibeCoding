# PRD — Pokémon 3D Arena（個人版寶可夢卡片收藏 × 3D 對戰網站）

> 版本：v1.0 ｜ 日期：2026-07-13 ｜ 狀態：待審核
> 定位：**純個人使用**的 side project（不公開、不商業化），因此可自由使用社群素材與任天堂 IP 相關資源。

---

## 1. 產品願景

把「我的實體寶可夢卡片收藏」變成一個活的 3D 世界：

1. **拍卡即入庫** — 用手機/相機拍下實體卡片，AI 自動辨識是哪張卡、查出目前市價，存入收藏。
2. **寶可夢中心收藏館** — 在一個 3D 寶可夢中心場景裡瀏覽所有收藏卡片與寶可夢。
3. **派出寶可夢實時對戰** — 從收藏中派出寶可夢，用方向鍵在歷代聯盟賽 3D 戰場上進行即時操作對戰（vs AI）。
4. **三種視覺世代切換** — 同一隻寶可夢可以在「點陣 → 動畫 → 現代 3D」三種畫風之間即時切換，重現寶可夢 25 年的視覺演進。

## 2. 目標用戶

- **就是我自己**（單一用戶）。有實體卡片收藏、熟悉寶可夢世界觀、想要一個好看好玩的個人收藏管理 + 對戰遊戲。
- 不需要帳號系統、多用戶權限、社群功能。

## 3. 核心功能

### F1. 卡片掃描與查價（Card Scanner）

| 項目 | 內容 |
|---|---|
| 輸入 | 手機拍照上傳（或桌機 webcam / 拖曳圖片） |
| 辨識 | Claude Vision API 讀取卡面：卡名 + 右下角編號（如 `025/193`）+ 系列符號 |
| 比對 | 以「卡名 + collector number + set 總數」查 Pokémon TCG API（pokemontcg.io v2），取得唯一卡片 |
| 查價 | 卡片物件內建 TCGplayer 市價（low / mid / high / market，近每日更新），以美金顯示 |
| 入庫 | 確認畫面（顯示官方卡圖 vs 我拍的照片並列）→ 存入收藏 DB，保留我拍的原始照片 |
| 容錯 | 辨識失敗 → 手動搜尋卡名補救；Vision 可能誤讀編號 → 以 API 回傳卡名交叉驗證 |

**驗收標準**：一張光線正常的英文卡，從拍照到顯示市價 < 15 秒；辨識正確率 > 90%（反光卡除外）。

### F2. 收藏頁 — 寶可夢中心場景（Collection / Pokémon Center）

- 3D 寶可夢中心室內場景：經典紅色屋頂大廳、櫃台（喬伊小姐位置）、治療機、電腦區。
- 收藏卡片以「卡片牆 / 展示櫃」形式陳列在場景中，可用滑鼠/方向鍵走動或環繞視角瀏覽。
- 點擊卡片 → 卡片放大檢視（3D 翻轉卡片、閃卡光澤 shader），顯示：卡片資訊、目前市價、價格歷史（每次刷新記錄一筆）、我拍的照片。
- 收藏總覽 HUD：總卡數、總市值（美金）、最貴的卡。
- 從卡片可「召喚」對應寶可夢（卡片 → 全國圖鑑編號 → 寶可夢實體）在場景中顯示。

**驗收標準**：50 張卡的收藏在 M 系列 Mac 上維持 60fps；卡片牆載入 < 3 秒。

### F3. 主畫面 — 實時 3D 對戰（Battle）

- **出戰選擇**：從收藏擁有的寶可夢中選擇 1–3 隻組隊（卡片沒有的寶可夢不能派出）。
- **戰場選擇**：一至八世代聯盟賽場地，共 8 個主題競技場（詳見 DESIGN.md 場景規格）：
  1. Gen 1 石英高原（火焰聖火 + 草/岩/水/冰四場地）
  2. Gen 2 白銀山銀色大會（機械旋轉場地 + 雪山背景）
  3. Gen 3 彩幽市大會（戰鬥中場地變換 + 海島瀑布）
  4. Gen 4 鈴蘭大會（哥德教堂 + 夜間煙火）
  5. Gen 5 因幡大會/合眾聯盟（夜間山頂四塔 + 聚光燈）
  6. Gen 6 密阿雷大會（六邊形體育場 + 元素機關）
  7. Gen 7 瑪納羅大會（海上露天球場 + 熱帶天空）
  8. Gen 8 宮門體育場（足球場式 + 雷射燈光秀 + 極巨化氛圍）
- **操作方式**（即時制，非回合制）：
  - 方向鍵：移動自己的寶可夢
  - Z / Space：招式 1（普攻，短冷卻）
  - X：招式 2（強攻，長冷卻）
  - C：衝刺 / 閃避
  - V：換上下一隻寶可夢
- **對手**：AI 控制（狀態機：接近 → 攻擊 → 拉開距離 → 換人），三段難度。
- **傷害計算**：採用 Pokémon 官方公式（@smogon/calc），保留屬性相剋、本系加成，轉換為即時制的 HP 消耗。
- **勝負**：一方全部寶可夢 HP 歸零；勝利結算畫面（經驗值/戰績純紀錄用）。

**驗收標準**：對戰全程 60fps（單場景 2 隻寶可夢 + 特效）；按鍵到動作反應 < 100ms；屬性相剋倍率與正作一致。

### F4. 視覺世代切換（Style Switcher）

全站級的畫風切換（設定面板 + 對戰中快捷鍵 Tab），三檔：

| 模式 | 寶可夢呈現 | 素材來源 |
|---|---|---|
| **點陣（Pixel）** | Gen 1–5 像素 sprite 貼片（billboard），復古濾鏡（可選 scanline） | Pokémon Showdown `gen1`–`gen5` sprites |
| **動畫（Animated）** | 動態 GIF sprite 貼片（呼吸、待機動作） | Showdown `gen5ani` / `ani` 動態圖 |
| **現代 3D（Modern）** | 完整 3D 模型（glTF），骨架動畫（若該模型有） | Pokemon-3D-api GLB 合集（1,300+ 模型） |

- 切換是即時的，不重載場景；場景本身（競技場、寶可夢中心）維持 3D，只切換「寶可夢與特效的呈現層」。
- 點陣/動畫模式下 UI 同步切換為像素字體風格；現代模式為乾淨的現代 UI。

**驗收標準**：三種模式切換 < 1 秒（素材預載）；同一隻寶可夢三種模式皆有對應素材（缺料時 fallback 到官方立繪）。

## 4. 用戶流程（Happy Path）

```
首次使用
  └─ 拍第一張卡 → 辨識成功 → 看到市價 → 入庫
        └─ 進入寶可夢中心 → 看到卡片上牆 → 召喚寶可夢
              └─ 回主畫面 → 選 1 隻出戰 → 選 Gen 8 宮門體育場
                    └─ 方向鍵對戰 AI → 獲勝 → 戰績記錄
                          └─ 按 Tab 切到點陣模式再打一場（情懷）
```

## 5. 範圍界定

### MVP（Phase 1）
- F1 卡片掃描 + 查價 + 入庫（美金/TCGplayer）
- F2 寶可夢中心收藏場景（基本版：卡片牆 + 檢視）
- F3 對戰：**1v1、vs AI、先做 2 個戰場**（Gen 1 石英高原 + Gen 8 宮門體育場，一舊一新對比最強）
- F4 畫風切換：先做 **點陣 ↔ 現代 3D** 兩檔

### Phase 2
- 其餘 6 個世代戰場
- 動畫（GIF）畫風檔
- 3v3 換人對戰、閃卡 shader、價格歷史圖表
- 寶可夢叫聲（PokéAPI cries）與各世代戰鬥 BGM

### Phase 3（Nice to have）
- 場地互動機制（Gen 2 機械換場、Gen 3 戰鬥中換場地作為對戰事件）
- 極巨化演出（Gen 8 場景限定的視覺事件）
- 日文卡辨識

### 非目標（Non-goals）
- ❌ 線上多人對戰（明確延後，架構預留但不實作）
- ❌ 帳號系統 / 多用戶
- ❌ 公開部署、商業化（版權素材限個人本機使用）
- ❌ 完整還原正作回合制對戰邏輯（我們是即時制動作對戰）
- ❌ 手機 App（響應式網頁即可，拍照走網頁上傳）

## 6. 成功指標（個人專案版）

- 我實際把 ≥ 30 張自己的卡入庫，且辨識體驗不煩躁
- 每週至少想打開來玩一次對戰
- 三種畫風切換的「哇」時刻成立（給朋友看會驚呼）

## 7. 風險與對策

| 風險 | 影響 | 對策 |
|---|---|---|
| pokemontcg.io 進入維護模式（已併入 Scrydex） | 查價斷炊 | 資料入庫時快取全部卡片資料+價格；備援：TCGdex（無價格）、Scrydex 免費層 |
| 3D 模型無動畫（多數 rip 只有 bind pose） | 對戰僵硬 | 用程式化動畫補（位移、squash & stretch、受擊閃白）；點陣/動畫模式天然有動態 |
| GIF 在 Three.js 無原生播放 | 動畫模式做不出來 | gifuct-js 解碼 → CanvasTexture 逐幀更新（已驗證可行的社群方案） |
| 反光/閃卡辨識失敗 | 入庫卡關 | 手動搜尋 fallback + 拍攝引導（避開直射光） |
| 素材版權 | 法律風險 | 嚴格限個人本機/私有部署，不公開網址、不放 GitHub public repo |
